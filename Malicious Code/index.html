# Security Control Security Requirement CWE
1 Code Integrity Controls Verify that a code analysis tool is in use that can detect potentially malicious code, such as time functions, unsafe file operations and network connections. CWE-749
2 No sensitive data in source Verify that the application source code and third party libraries do not contain unauthorized phone home or data collection capabilities. Where such functionality exists, obtain the user's permission for it to operate before collecting any data. CWE-359
3 No sensitive data in source Verify that the application does not ask for unnecessary or excessive permissions to privacy related features or sensors, such as contacts, cameras, microphones, or location. CWE-272
4 No sensitive data in source Verify that the application source code and third party libraries do not contain back doors, such as hard-coded or additional undocumented accounts or keys, code obfuscation, undocumented binary blobs, rootkits, or anti-debugging, insecure debugging features, or otherwise out of date, insecure, or hidden functionality that could be used maliciously if discovered. CWE-507
5 No sensitive data in source Verify that the application source code and third party libraries do not contain time bombs by searching for date and time related functions. CWE-511
6 No sensitive data in source Verify that the application source code and third party libraries do not contain malicious code, such as salami attacks, logic bypasses, or logic bombs. CWE-511
7 No sensitive data in source Verify that the application source code and third party libraries do not contain Easter eggs or any other potentially unwanted functionality. CWE-507
8 Deployed Application Integrity Controls Verify that if the application has a client or server auto-update feature, updates should be obtained over secure channels and digitally signed. The update code must validate the digital signature of the update before installing or executing the update. CWE-16
9 Deployed Application Integrity Controls Verify that the application employs integrity protections, such as code signing or subresource integrity. The application must not load or execute code from untrusted sources, such as loading includes, modules, plugins, code, or libraries from untrusted sources or the Internet. CWE-353
10 Deployed Application Integrity Controls Verify that the application has protection from subdomain takeovers if the application relies upon DNS entries or DNS subdomains, such as expired domain names, out of date DNS pointers or CNAMEs, expired projects at public source code repos, or transient cloud APIs, serverless functions, or storage buckets (autogen-bucket-id.cloud.example.com) or similar. Protections can include ensuring that DNS names used by applications are regularly checked for expiry or change. CWE-350
