# Security Control Security Requirement CWE
1 Have proper data classification Verify that regulated private data is stored encrypted while at rest, such as Personally Identifiable Information (PII), sensitive personal information, or data assessed likely to be subject to EU's GDPR. CWE-311
2 Have proper data classification Verify that regulated health data is stored encrypted while at rest, such as medical records, medical device details, or de-anonymized research records. CWE-311
3 Have proper data classification Verify that regulated financial data is stored encrypted while at rest, such as financial accounts, defaults or credit history, tax records, pay history, beneficiaries, or de-anonymized market or research records. CWE-311
4 Usage of strong cyptographic algorithms Verify that all cryptographic modules fail securely, and errors are handled in a way that does not enable Padding Oracle attacks. CWE-310
5 Usage of strong cyptographic algorithms Verify that industry proven or government approved cryptographic algorithms, modes, and libraries are used, instead of custom coded cryptography. CWE-327
6 Usage of strong cyptographic algorithms Verify that encryption initialization vector, cipher configuration, and block modes are configured securely using the latest advice. CWE-326
7 Usage of strong cyptographic algorithms Verify that random number, encryption or hashing algorithms, key lengths, rounds, ciphers or modes, can be reconfigured, upgraded, or swapped at any time, to protect against cryptographic breaks. CWE-326
8 Usage of strong cyptographic algorithms Verify that known insecure block modes (i.e. ECB, etc.), padding modes (i.e. PKCS#1 v1.5, etc.), ciphers with small block sizes (i.e. Triple-DES, Blowfish, etc.), and weak hashing algorithms (i.e. MD5, SHA1, etc.) are not used unless required for backwards compatibility. CWE-326
9 Usage of strong cyptographic algorithms Verify that nonces, initialization vectors, and other single use numbers must not be used more than once with a given encryption key. The method of generation must be appropriate for the algorithm being used. CWE-326
10 Usage of strong cyptographic algorithms Verify that encrypted data is authenticated via signatures, authenticated cipher modes, or HMAC to ensure that ciphertext is not altered by an unauthorized party. CWE-326
11 Usage of strong cyptographic algorithms Verify that all cryptographic operations are constant-time, with no 'short-circuit' operations in comparisons, calculations, or returns, to avoid leaking information. CWE-385
12 Generate random values Verify that all random numbers, random file names, random GUIDs, and random strings are generated using the cryptographic module's approved cryptographically secure random number generator when these random values are intended to be not guessable by an attacker. CWE-338
13 Generate random values Verify that random GUIDs are created using the GUID v4 algorithm, and a Cryptographically-secure Pseudo-random Number Generator (CSPRNG). GUIDs created using other pseudo-random number generators may be predictable. CWE-338
14 Generate random values Verify that random numbers are created with proper entropy even when the application is under heavy load, or that the application degrades gracefully in such circumstances. CWE-338
15 Verify the secret management is in place Verify that a secrets management solution such as a key vault is used to securely create, store, control access to and destroy secrets CWE-798
16 Verify the secret management is in place Verify that key material is not exposed to the application but instead uses an isolated security module like a vault for cryptographic operations. CWE-320
