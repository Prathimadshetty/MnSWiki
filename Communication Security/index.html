# Security Control Security Requirement CWE
1 Communications Security Requirements Verify that secured TLS is used for all client connectivity, and does not fall back to insecure or unencrypted protocols. CWE-319
2 Communications Security Requirements Verify using online or up to date TLS testing tools that only strong algorithms, ciphers, and protocols are enabled, with the strongest algorithms and ciphers set as preferred. CWE-326
3 Communications Security Requirements Verify that old versions of SSL and TLS protocols, algorithms, ciphers, and configuration are disabled, such as SSLv2, SSLv3, or TLS 1.0 and TLS 1.1. The latest version of TLS should be the preferred cipher suite. CWE-326
4 Server Communications Security Requirements Verify that connections to and from the server use trusted TLS certificates. Where internally generated or self-signed certificates are used, the server must be configured to only trust specific internal CAs and specific self-signed certificates. All others should be rejected. CWE-295
5 Server Communications Security Requirements Verify that encrypted communications such as TLS is used for all inbound and outbound connections, including for management ports, monitoring, authentication, API, or web service calls, database, cloud, serverless, mainframe, external, and partner connections. The server must not fall back to insecure or unencrypted protocols. CWE-319
6 Server Communications Security Requirements Verify that all encrypted connections to external systems that involve sensitive information or functions are authenticated. CWE-287
7 Server Communications Security Requirements Verify that proper certification revocation, such as Online Certificate Status Protocol (OCSP) Stapling, is enabled and configured. CWE-299
8 Server Communications Security Requirements Verify that backend TLS connection failures are logged. CWE-544
